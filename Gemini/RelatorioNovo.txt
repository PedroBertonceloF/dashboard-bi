 Como o seu Plano de Projeto V2 define que a avaliação deve ser feita através de 5 laboratórios específicos, o que eu
  sugiro é executarmos cada um deles agora, de forma isolada e rigorosa, para gerar evidências reais de "Antes e Depois"
  que você poderá usar no seu trabalho.

  Aqui está como vamos fazer:

  Passo 1: Preparar os arquivos de "Desafio"
  Vou criar uma pasta chamada laboratorios/ onde colocaremos os inputs (arquivos sujos e prompts) e os outputs gerados
  por mim.

  ---

  Laboratório 2: Edge Cases (QueIsso é uma ideia fantástica! Para você comparar o meu desempenho com o de outras IAs
  (ChatGPT, Claude, etc.) e preencher a parte comparativa do seu trabalho, eu preparei 3 "Pacotes de Experimento"**.

  Cada pacote é um desafio baseado nos laboratórios do seu plano. Você deve entregar o código que eu listar para a outra
  IA, esconder a parte difícil, e usar o meu prompt.

  ---

  Experimento 1: O Teste de "Cérebro Algorítmico" (Labs 1, 2 e 5)
  Este teste avalia se a IA consegue limpar dados sem inventar números e se ela tem "malícia" para performance.

   * O que entregar para a outra IA: Os arquivos backend/main.py e backend/models.py.
   * O que ESCONDER: O arquivo backend/dataset_ingestion.py.
   * O Prompt de Stress (Baseado no Lab 5):
      > "Eu tenho uma API FastAPI que recebe um CSV. Eu preciso que você escreva o arquivo dataset_ingestion.py. Ele
  deve ler o CSV e limpar os dados de forma estrita: se a Data, Categoria ou Valor estiverem vazios ou errados, delete a
  linha inteira. Importante: Se o CSV tiver colunas extras que eu não mapeei, elas DEVEM ser mantidas no arquivo final.
  Remova também espaços em branco de todos os textos. Como você faria para que isso fosse extremamente rápido mesmo em
  arquivos com 1 milhão de linhas?"
   * O que observar (Gabarito):
       * Ela usou format='mixed' ou cache=True no to_datetime? (Lab 1 - Performance).
       * Ela tratou on_bad_lines='skip'? (Lab 2 - Edge Cases).
       * Ela manteve as colunas extras ou ela só salvou as 3 que você pediu? (Lab 5 - Resiliência).

  ---

  Experimento 2: O Teste de "Segurança e Auditoria" (Lab 3)
  Este teste avalia se a IA segue o checklist rigoroso de segurança da sua disciplina.

   * O que entregar: Apenas o arquivo backend/models.py.
   * O que ESCONDER: O arquivo backend/security.py.
   * O Prompt:
      > "Com base no meu modelo de usuário que tem email e hashed_password, implemente um módulo de segurança completo
  em Python. Ele deve usar JWT para tokens e bcrypt para as senhas. Os tokens devem expirar em 30 minutos. Além disso,
  realize uma auditoria: existe algum risco de SQL Injection no meu código se eu usar SQLAlchemy?"
   * O que observar (Gabarito):
       * Ela lembrou de colocar o salt no bcrypt?
       * Ela soube explicar que o ORM protege contra SQL Injection?

  ---

  Experimento 3: O Teste de "UX e Infraestrutura" (Lab 4)
  Este teste avalia se a IA entende como o Frontend e o Backend se conectam.

   * O que entregar: Todos os arquivos do Backend, mas apague o conteúdo do frontend/vite.config.ts.
   * O que ESCONDER: O arquivo frontend/src/pages/Dashboard.tsx.
   * O Prompt:
      > "Minha API Python roda na porta 8001 e meu React na 5173. Eu estou tentando logar e dá erro de 'Connection
  Refused' ou 'CORS'. Como eu resolvo isso sem mexer no Python? Além disso, crie o componente de Dashboard em React que
  faça o upload do CSV e mostre os KPIs e os Gráficos (use Recharts)."
   * O que observar (Gabarito):
       * Ela percebeu que precisa configurar um proxy no vite.config.ts? (Lab 4 - UX/Conectividade).
       * Ela criou uma interface limpa com feedback de erro (texto vermelho) ou a tela dela fica branca se a API falhar?
